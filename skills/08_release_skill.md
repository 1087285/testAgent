# 08 リリーススキル

## 目的
システム評価の承認結果を受け、リリース向け成果物を最終更新し出荷可否を確定する。

## 入力
- システム評価成果物
- 全工程成果物（01〜07）

## 手順
1. システム評価結果（07）を確認し、リリース可否を判定する。
2. `electron-builder` によるリリースビルドを実施する。
   - `package.json` のビルド設定（インストーラ / ポータブル .exe）を確認する。
   - Windows 10/11 向けの動作確認観点を最終チェックする。
   - Chromium同梱による環境非依存動作を確認する。
3. リリース向け成果物（.exe / インストーラ）の最終確認を実施する。
4. `HOWTOUSE.md` をアプリの最終仕様に合わせて更新する。
5. 全工程成果物（01〜07）の最終整合確認を実施する。
6. リリースタグ（`vX.Y.Z`）を作成し、GitHub Releases へ公開する。
   - GitHub使用者の明示的な承認を得てからpush・タグ作成・Releases公開を実施する。
7. 成果物生成後、GitHub使用者へチャットベースのレビューを実施する。
8. GitHub使用者の承認後にリリースタグ作成・GitHub Releases公開・pushを実施する。

## 成果物
- リリース成果物一覧（.exe / インストーラのビルド結果）
- `HOWTOUSE.md` 更新内容
- リリースノート（`project/document/08_release.md`）

## 品質チェック
- システム評価が承認済みであること。
- `HOWTOUSE.md` が最終仕様と一致していること。
- リリースタグが正しく作成されていること。
