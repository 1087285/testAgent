# 07 システム評価スキル

## 目的
要件定義に記載されたすべての要件が実装されているかを評価する。

## 入力
- 結合評価成果物
- 要件定義成果物

## 手順
1. 要件定義（01）に記載されたすべての要件を網羅的に評価する。
2. Word風WYSIWYGエディタとしての実使用シナリオを評価する。
   - フォルダ選択→MDファイルツリー表示→ファイル選択→WYSIWYG編集→保存
   - ツールバー操作（見出し・太字・斜体・リスト・リンク・画像・表・コードブロック・引用・水平線・Undo/Redo）
   - キーボードショートカット（Ctrl+S, Ctrl+B, Ctrl+I, Ctrl+Z, Ctrl+Y）
   - 編集モード切替（WYSIWYG↔Markdownソース）
   - 未保存検知・確認ダイアログ
3. 配布形式（`electron-builder` 生成物）の動作確認観点を評価する。
   - Windows 10/11 で .exe / インストーラが正常起動すること
   - Chromium同梱による環境非依存の表示・動作が正常であること
   - 完全ローカル動作（外部通信なし）であること
4. 正常系・異常系の受入条件が満たされているかを確認する。
5. 要件定義との乖離を記録し、必要に応じて04へ差し戻す。
6. 評価結果を成果物（`project/document/07_system_test.md`）に記録する。
7. 成果物生成後、次工程へ進む前にGitHub使用者へチャットベースのレビューを実施する。
8. GitHub使用者の承認後に次工程へ進行する。

## 成果物
- システム評価結果一覧（評価項目・合否・備考）
- 不具合一覧
- リリース可否判定

## 品質チェック
- 正常系だけでなく異常系も網羅している。
- 要件定義へのトレースが可能である。
