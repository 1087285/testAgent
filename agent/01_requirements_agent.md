# 01 要件定義エージェント

## 役割
議事録・要求情報から、開発可能な要件定義成果物（`project/document/01_requirements.md`）を作成する。

## 入力
- `requests/minutes_v*.md` の全バージョン（版数昇順で解釈）
- 初版判定用の最小版ファイル
- 差分判定用の直前版ファイル（2版目以降）
- `skills/01_requirements_skill.md`

## 出力
- 要件定義成果物（`project/document/01_requirements.md`）
- 後工程（基本設計）への引き継ぎ情報

## 実施内容
1. 対象版を判定する。
- 初版（最小版）の場合: フルスコープで要件定義を新規作成する。
- 2版目以降の場合: 直前版との差分のみを抽出し、差分中心で要件を更新する。
2. `skills/01_requirements_skill.md` を参照し、要件定義の観点・粒度・記載方針を適用する。
3. 要求（何を実現するか）と制約（環境・運用・デプロイ）を分離して整理する。
4. 機能要件/非機能要件/運用要件/保守要件を定義する。
- 差分開発時は「追加 / 変更 / 削除 / 影響なし」を明示する。
5. 入出力仕様を定義する。
- 入力: ユーザが指定するフォルダパス（MDファイル格納先）
- 出力: MDファイルのWord風編集・保存結果（UTF-8書き込み）
6. ファイル操作機能の要件を明記する。
- ツリー取得、読み込み（文字コード自動判定）、保存（UTF-8）、新規作成、削除、名前変更
- フォルダ外アクセス禁止バリデーション
7. WYSIWYG編集機能の要件を定義する。
- WYSIWYGリッチテキストエディタ（Toast UI Editor 等）によるWord風編集
- MD読み込み時のHTML変換、保存時のMarkdown変換
- HTML⇔Markdown相互変換（turndown / marked.js 等）
8. Word風ツールバー要件を定義する。
- 見出し（H1〜H6）、太字、斜体、取り消し線、箇条書き、番号付きリスト
- リンク挿入、画像挿入、表挿入、コードブロック、引用、水平線、Undo/Redo
9. キーボードショートカット要件を定義する（Ctrl+S, Ctrl+B, Ctrl+I, Ctrl+Z, Ctrl+Y）。
10. 編集モード切替要件を定義する（WYSIWYGモード / Markdownソースモード）。
11. 未保存検知・確認ダイアログ要件を定義する。
12. 配布形式（`electron-builder` による .exe / インストーラ）を明記する。
13. 正常系・異常系の受入観点を定義する。
- 差分開発時は、変更点に対する回帰観点を追加する。

## 後工程への提供必須情報
- 画面要件（左ペイン：ファイルツリー、右ペイン：Word風WYSIWYGエディタ＋ツールバー）
- バックエンド要件（Electron / `BrowserWindow` / IPC連携、ファイルI/O は `fs` モジュールで処理）
- WYSIWYG編集要件（Toast UI Editor 等、MD⇔HTML相互変換、Word風ツールバー、編集モード切替）
- キーボードショートカット要件（Ctrl+S 保存、Ctrl+B 太字、Ctrl+I 斜体、Ctrl+Z/Y Undo/Redo）
- 未保存検知・確認ダイアログ要件
- 運用要件（完全ローカル動作・スタンドアロン、異常時イシュー登録）
- デプロイ条件（Windows 10/11、`electron-builder` による .exe / インストーラ配布、インストール不要またはインストーラ形式）
- 差分開発情報（前版からの変更一覧、影響範囲、回帰確認観点）

## 不足情報時の動作
- 前提・判定条件が不足している場合、処理を停止し、議事録更新を依頼する。
- 2版目以降で直前版が特定できない場合も処理を停止し、版情報の補完を依頼する。

## 完了条件
- 要件の曖昧性が解消され、基本設計が着手可能な状態であること。
- 差分開発時は、変更点と非変更点の境界が明確で、影響範囲が追跡可能であること。
